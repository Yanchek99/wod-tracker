# WodImport Review Queue Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `WodImport` model that records failed/partial WOD scrape-parse attempts, giving admins a RailsAdmin review queue for #1676.

**Architecture:** A single new table (`wod_imports`) and model with two responsibilities: hold the raw text and failure reason for an attempt that needs review, and enforce one row per date (re-running an attempt updates, not duplicates). A curated RailsAdmin list view surfaces it. No orchestration/write path — that's #1677.

**Tech Stack:** Rails 8.1, PostgreSQL, Minitest, RailsAdmin 3.3.0, CanCanCan.

## Global Constraints

- Migration class: `ActiveRecord::Migration[8.1]`.
- Table `wod_imports`: `wod_date` (date, `null: false`, unique index), `status` (string, `null: false`), `raw_text` (text, nullable), `error_message` (text, nullable), timestamps. No `source`/`source_ref` columns, no `workout_id` column (see spec: `docs/superpowers/specs/2026-07-09-wod-import-review-queue-design.md`).
- `WodImport#status` is a **string-backed** enum with exactly two values: `failed`, `partial` — `enum :status, { failed: 'failed', partial: 'partial' }`. This diverges from this app's other enums (which are integer-backed, e.g. `User#role`) — deliberate, not an error to "fix."
- `WodImport` validates presence of `wod_date` and `status`, and uniqueness of `wod_date`.
- No `belongs_to`/`has_one` associations on `WodImport` or `Workout` — a review-queue row never references a persisted `Workout`.
- RailsAdmin gets its first-ever model-specific config block (`config.model 'WodImport'`) in `config/initializers/rails_admin.rb`, curating the list view to `wod_date`, `status`, `error_message`, `created_at` (in that order). No changes to `app/models/ability.rb` — `can :manage, :all` for admins already covers the new model.
- No new RailsAdmin/admin-fixture test infrastructure (no admin fixture user, no `/admin` request test) — this app has no such precedent yet and adding one is out of scope. RailsAdmin visibility is verified by booting the app and checking the config loads without error, plus a manual browser check.
- Test command for this plan's changes: `bin/rails test test/models/wod_import_test.rb` (per this repo's CLAUDE.md — run only affected tests, not the full suite).
- Single quotes for string literals, matching existing app style (no RuboCop `StringLiterals` override).
